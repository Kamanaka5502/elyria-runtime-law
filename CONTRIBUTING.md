## Contribution posture

Public contributions are not accepted unless explicitly invited in writing.

The repository exists for controlled public review of category framing, consequence-admission vocabulary, and protected runtime-law positioning.

## Permitted public feedback

Public feedback may address:

```text
documentation clarity
public wording errors
broken links
protected-scope ambiguity
unsafe category interpretation
attribution corrections
```

## Not accepted publicly

Do not submit:

```text
production runtime logic
private evaluator logic
client-specific corridors
customer policies
customer evidence
protected schemas
receipt-generation internals
replay-verification internals
production adapters
commercial implementation code
regulated deployment material
```

## Review rule

All proposed changes must preserve the boundary:

```text
public category clarity without exposing protected production substrate
```
