```text

```

## Protected scope

This repository does not publish:

```text
private runtime law

client-specific policies

production adapters
private schemas

receipt-generation internals

```

## Boundary rule

Public material may explain the category:

```text
consequence admission before effect
EXECUTE is the only outcome that may bind consequence
REFUSE / ESCALATE / HALT must fail closed
receipt and replay must support proof
```

Public material must not disclose the protected machinery that implements production enforcement.

## Review posture

The repo is safe for public review because it is intentionally bounded.

Anyone reviewing this repository should understand the category, not be able to recreate the protected production substrate.

## Non-reduction rule

Elyria Runtime Law must not be collapsed into:

```text
a dashboard
a monitoring layer
a policy checklist
a compliance wrapper
a post-event audit tool
a generic authorization layer
```

Those surfaces may supply inputs. They do not decide whether consequence may bind.
